# 产品原则

## 1.0 原则

1.0 是本地优先/用户端优先的静态 Web App：

- 用户数据只在用户端保存。
- 不要求注册账号。
- 不上传薪资、Offer、公司、职级、城市、家庭偏好等敏感信息。
- 项目只提供计算公式、默认假设、UI 和可选的公开第三方数据。
- 第三方数据可以通过 GitHub Pages、静态 JSON、版本化文件等方式接入。

## 后端上线前置条件

只有在以下问题考虑清楚后，才进入后端版本：

- 用户授权
- 数据加密
- 脱敏和匿名化
- 删除和导出机制
- 数据最小化
- 权限隔离
- 日志脱敏
- 样本量保护，避免小样本反推个人
- 合规免责声明

## 计算边界

### 可计算

能较稳定折算成钱的项目进入真实年包：现金、税、奖金、股权、期权、五险一金、工作必要支出、通勤现金成本、住房差异、搬家成本、餐饮自付成本变化。

### 只提醒

很重要但主观权重差异大的项目默认只提醒，不进入主数字：通勤时间、离家远近、团队氛围、高压程度、成长性、简历增益、业务兴趣、家庭便利性。

用户可以选择手动给这些软因素输入权重或金额，但产品默认不替用户做价值判断。

## 福利抵扣原则

公司福利不是自动加钱。更稳妥的模型是：

- 工作导致的必要支出先作为扣减项。
- 公司福利优先抵扣这些扣减项。
- 抵扣到 0 为止，除非福利可以直接变现，否则不继续作为收入加到年包里。

例如免费班车抵扣通勤费，最多把通勤现金成本降到 0；它不是额外收入。
